Computer Science Create

Weekly Log

Week 1

I read over the brief and researched several embedded systems to take inspiration for my own embedded system. I decided to go down the route of reducing screentime in order to improve wellbeing by using a countdown timer to track user screentime.

Week 2

I finalized this system as my idea and began to work vigorously on it. I worked on my investigation to clearly lay out what I need ed to do for my project to meet all of the requirements.

Week 3

Throughout week 3, I completed my investigation and began to work on my plan and design to mind map what my system will do, how it will carry out its functions and clearly display how my system operates using flowcharts and thinkercad to make an architect diagram.

Week 4

In my 4th week, I completed my plan and design, neatly formatting it and I listed out what I needed to do for my what ifs and analysis component.

Week 5

I began working on microsoft makecode to program what my microbit would do for my system, setting the functions for each of the microbit’s buttons and inputs for my system, such as how to set screentime for the user using the microbit and how to meet the advanced requirements using data sent by the microbit.

Week 6

I had finished the microbit code in makecode and began to work on the python code to send microbit data to firebase. I added inputs to take age, name, and wellbeing level for parameters to answer my what if questions.

Week 7

I continued to work on my code for sending data to firebase and making sure that everything that I wanted to send was sending correctly.

Week 8

I spent week 8 working on reading data from firebase and drawing conclusions based on a case study to give the user feedback on the impact of their screentime on their mental health, which meets my analysis component.

Week 9

I spent the week making a graph that used the data for hours and minutes from firebase and converted minutes into a percentage of hours to be displayed on the graph.

Week 10

Week 10 was spent finishing up the code for the graph. I also started working on a review scale for one of my what if questions.

Week 11

I finished my review scale and made a program to offer user feedback on activities they should do based on age, screentime and free time.

Week 12

I spent the final week filming and editing the video, as well as testing the code to make sure everything was working.

Problem Faced

The biggest problem that I faced whilst making my model was an issue I encountered when using Microsoft MakeCode to send data to Python, which would then send to Firebase. The issue that I encountered was that the data from the microbit wasn't being sent to Python properly, and to Firebase by extension. Data would sometimes either not send, or duplicate itself and send seemingly random values. The way I resolved this issue was if I made any changes to MakeCode, I would download it to the microbit and then disconnect the microbit from MakeCode, which allowed the data to send correctly.

Important Code

The most important piece of code for me was the code that I used to send data to firebase from my microbit through python, as without that code, data would not be stored anywhere for my model to use. It stores each character from my microbit data into a string until it reaches a certain character, where it converts certain characters into integers for my model to use as time, before emptying its values to be used again.

Evaluation

I believe that throughout the process of doing this project, I have achieved what I had intially set out to accomplished and created a model that has the purpose of improving wellbeing in some form. I believe I have achieved this as my model meets the basic requirements and advanced requirements that I set out to meet since the start of the project and meets the intended purpose of improving wellbeing through reducing screentime. In my opinion, it meets the requirements quite well, and my plan & design and investigation back up my claims as I believe that I have listed clearly everything that my model incorporates as well as its functions. It meets the basic requirements as it uses analogue and digital outputs and inputs, stores data in firebase through the use of python, and has an analysis component that runs a calculation which gives the user feedback based on their average screentime. My model also meets the advanced requirements through having 2 functioning programs to answer my what if questions as well as displaying information from firebase onto a graph. My Create I believe is also of high quality as it contains a weekly log, unit testing of some pieces of my code, a problem that I had faced, and an important piece of my code. The model meets the end requirements of the user through offering feedback to them on how to improve mental wellbeing and allowing them to give feedback on their mental wellbeing, from which the system can offer advice. If I were to make improvements upon my model, I would add more features to check how often the program was used, how consistent the usage was, as well as adding a system to assign daily goals to the user and congratulate the user upon accomplishing these tasks.
